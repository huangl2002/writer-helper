# 开发报告：v0.3.1 代码审查修复

## 日期：2026-05-30

## 版本：v0.3.1

## 概述

基于全面的代码审查（9 个角度 × 8 个候选），发现并修复了 6 个确认 Bug 和 5 个代码质量问题。

---

## 确认 Bug 修复

### 1. handleFormat 排版函数三重 bug (WritingEditor.tsx)

**发现：** 代码审查 Angle A（逐行扫描）

- **Bug 1 — 重复内容：** `doc.descendants` 遍历所有后代节点，列表/引用块内的段落被容器 `toJSON()` 包含又被独立 `push`，导致重复
- **Bug 2 — 标记丢失：** 段落处理器只读 `textContent` 重建为 `{ type: "text", text }`，丢失加粗/斜体/链接等所有内联标记
- **Bug 3 — 引号方向：** 直双引号统一映射为右弯引号，中文排版需要左右配对

**修复：** 完全重写函数——用 `editor.getJSON()` + 递归 walk() 就地替换文本节点的标点，再在顶层添加段落间空行，保留所有原有结构和标记

### 2. QuickSearch 搜索原始 JSON (QuickSearch.tsx)

**发现：** `ch.content_json?.toLowerCase().includes(lower)` 搜索的是 ProseMirror 原始 JSON（如 `{"type":"doc","content":[{"type":"paragraph"...}`），搜索 "type" 或 "text" 会匹配所有章节

**修复：** 用 `extractPlainText()` 工具函数提取纯文本后再搜索

### 3. QuickSearch 被 store 更新清空 (QuickSearch.tsx)

**发现：** useEffect 依赖 `[visible, chapters, works]`，自动保存后 store 更新触发 effect 重新执行，清空用户正在输入的搜索词

**修复：** 用 `useMemo` 预计算 enriched 列表，用 ref 标记初始化状态，避免重复重建

### 4. 崩溃缓冲区未清理 (WritingEditor.tsx)

**发现：** 注释说"clean unmount 时清除 crash buffer"，但 cleanup 函数只清除了 interval，未删 localStorage

**修复：** cleanup 中添加 `localStorage.removeItem(CRASH_BUFFER_KEY)`

### 5. URL 协议检查大小写 (EditorToolbar.tsx)

**发现：** `linkUrl.startsWith("http")` 对 "HTTP://" 返回 false，产生 "https://HTTP://..." 畸形 URL

**修复：** 改用 `/^https?:\/\//i.test(linkUrl)` 大小写不敏感

### 6. AI 流式持久化性能 (AiChat.tsx)

**发现：** SSE 流式传输中每收到一个 chunk 就执行 `JSON.stringify` + `localStorage.setItem`，50-100 次同步写阻塞主线程

**修复：** 改用 debounced 持久化（3 秒无新 chunk 才写入）

## 代码质量修复

### 7. StatusBar 双计时器 (StatusBar.tsx)

**问题：** 两个独立的 `setInterval(1000)` 分别更新会话计时和时钟，导致两次渲染/秒

**修复：** 合并为单个 `tick()` 函数

### 8. StatusBar 动态导入 (StatusBar.tsx)

**问题：** `import("../../lib/db")` 动态导入浪费异步开销，且其他文件都用的静态导入

**修复：** 改为顶层 `import * as db`

### 9. 打字机按钮重复 (WritingEditor.tsx)

**问题：** 完全相同的 `<button>` JSX 在两个渲染分支各写一遍

**修复：** 提取为 `<TypewriterBtn />` 内部组件
