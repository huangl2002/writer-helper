| 维度 | 评估 |

# 第三部分：优化实施计划

## 第一阶段（1-2 周）：核心痛点修复

### 1.1 番茄钟桌面通知 + 自动循环
- 使用 Tauri notification API 发送桌面通知
- 番茄钟结束后自动进入休息/专注循环
- 全局状态追踪，跨页面显示计时器

### 1.2 崩溃恢复机制
- 每 10 秒把编辑器内容存到 `localStorage` 的 crash buffer
- 启动时检测 crash buffer，有内容则弹出恢复提示
- 正常退出时清除 crash buffer

### 1.3 AI 流式输出
- 将 `stream: false` 改为 `stream: true`
- 用 ReadableStream 逐块读取并追加到消息中
- 显示打字效果，降低等待焦虑

### 1.4 侧栏拖拽排序
- 为 WorkTree 的章节和卷加入简单的上下移动按钮（低成本方案）
- 后续可接入拖拽库实现真正的拖拽排序

### 1.5 大纲拖拽排序
- 大纲节点加"上移/下移"按钮
- 移动时调用 `db.moveOutline` 更新 sort_order 字段

## 第二阶段（3-4 周）：编辑器体验升级

### 2.1 打字机滚动模式
- 在编辑器设置中开启"打字机模式"
- 光标始终保持在视口 1/3 处
- 可开关，默认关闭

### 2.2 AI 编辑器集成
- 编辑器内选中文字 → 右键菜单增加"AI 续写""AI 润色""AI 扩写"
- 结果以 diff 形式展示
- 一键"采纳"替换选中内容或插入光标处

### 2.3 自定义对话框替换原生 prompt/confirm
- 创建一个 `ModalInput` 组件（标题 + 输入框 + 确认/取消）
- 创建一个 `ModalConfirm` 组件（标题 + 确认/取消）
- 替换全项目中的 prompt/confirm 调用

### 2.4 AI 上下文感知
- AI 请求自动附加：当前章节前 500 字 + 作品简介 + 当前大纲节点
- 用户可控制是否开启上下文注入

## 第三阶段（5-6 周）：功能增强

### 3.1 版本对比
- 选中两个快照，渲染为 diff（使用简单的逐行对比算法）
- 高亮新增/删除内容
- 支持逐段恢复

### 3.2 数据统计增强
- 添加近 30 天码字趋势折线图
- 添加作品维度统计
- 添加码字速度（字/小时）计算

### 3.3 角色出场追踪
- 在编辑器内容中自动检测角色名
- 记录角色→章节的出场关系
- 在角色卡片中显示出场列表

### 3.4 其他小修复
- HelperPanel 实时刷新（监听 store 变化）
- 单章导出放开 10 章限制
- AI 连接测试按钮
- AI 对话历史持久化
- DOCX 导出支持

# 第五部分：技术债务

| 债务 | 说明 |
|------|------|
| `AiChat` 中 `api_key_encrypted` 字段名误导 | 解密后仍用原字段名，容易误解，建议返回时重命名为 `api_key` |
| 无 TypeScript 严格模式 | `tsconfig.json` 中 `strict: false`，建议逐步启用 |
| 无单元测试 | 0 个测试文件，核心工具函数应补充测试 |
| Zustand store 过大 | 单一 store 包含所有状态，建议按功能拆分为多个 slice |
| 静态 `import()` 混用 | `db.ts` 被多文件静态 import 但也被 `StatusBar` 动态 import，打包警告 |

## 总结

**当前版本整体评价：** 功能完整度 75%，用户体验 65%，作为一个 V1 本地写作工具，核心写作流程可用，但打磨度不足。主要痛点在 AI 体验（无流式、无编辑器集成）、数据安全感（无崩溃恢复、无版本对比）、操作效率（无拖拽排序、原生弹窗）。优先修复 P0/P1 问题可大幅提升用户满意度。
