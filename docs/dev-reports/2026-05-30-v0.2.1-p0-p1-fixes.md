# 开发报告：v0.2.1 P0/P1 紧急修复

## 日期：2026-05-30

## 版本：v0.2.1

## 概述

基于 v0.2.0 全功能测试报告识别的 47 个问题，本轮优先修复了 5 个 P0（致命）和 3 个 P1（重要）问题。

---

## P0 修复

### 1. AI 流式输出 (B38)

**问题：** AI 请求使用 `stream: false`，用户需等待 10-30 秒才能看到完整回复，体验极差。

**修复：** 改用 `stream: true` + SSE 解析，通过 `ReadableStream` 逐 token 追加到消息列表。
- 用户立即看到 AI 开始输出，等待焦虑降为零
- 新增文件：无（修改 `AiChat.tsx`）

### 2. 崩溃恢复 (B6)

**问题：** 应用崩溃或意外退出时，未保存的编辑内容完全丢失。

**修复：** 每 10 秒将编辑器内容（章节 ID、标题、JSON 内容、时间戳）写入 `localStorage` 的 crash buffer。启动时检测 buffer，弹出琥珀色恢复横幅。
- 成功保存后自动清除 buffer，正常退出不会误触发
- 支持"恢复"和"放弃"两个操作

### 3. 大纲排序 (B25)

**问题：** 大纲节点按创建顺序排列，无法调整，需要"删了重建"来排序。

**修复：** 每个大纲节点添加 ↑ ↓ 按钮（hover 可见），通过 swap sort_order + `reorder_outlines` 命令实现同层排序。
- 新增依赖：无（复用已有 `reorder_outlines` 后端命令）

### 4. 番茄钟桌面通知 + 自动循环 (B31)

**问题：** 番茄钟结束时无任何提示，且需手动切换专注/休息。

**修复：**
- 使用 Web Notification API 发送桌面通知
- 专注结束自动进入休息，休息结束回到空闲（可手动 break）
- 首次使用时自动请求通知权限

### 5. 版本对比 (B35)

**问题：** 恢复历史版本前无法看到差异，只能"盲恢复"。

**修复：** 每个快照行新增"对比"按钮，选中两个后显示 LCS 差分视图。
- 新增行绿色高亮，删除行红色高亮，未变行正常显示
- 支持 Tiptap JSON 和纯文本两种格式的对比
- 新增文件：无（修改 `SnapshotPanel.tsx`，新增 `computeDiff` 函数）

## P1 修复

### 6. 自定义弹窗系统 (B1)

**问题：** 全项目使用浏览器原生 `prompt()` / `confirm()`，体验廉价。

**修复：**
- 新增 `Modal.tsx` 组件，提供 `ModalProvider` + `useModal()` hook
- 提供 `modalPrompt(title, defaultValue)` 和 `modalConfirm(title)` 两个 API
- 返回 Promise，与原生 API 签名兼容，替换成本极低
- 替换范围：WorkTree (6处)、CharacterList (1处)、OutlineTree (2处)、NotesList (1处)、SnapshotPanel (2处)、AiSettings (1处)，共 12 处
- 新增文件：`src/components/common/Modal.tsx`

### 7. AI 编辑器集成 (B39)

**问题：** 使用 AI 需切到 AI 页面、粘贴内容、选模式、发送——流程割裂。

**修复：** 编辑器中选中文字后，自动出现浮动工具栏。
- 4 个按钮：续写 / 润色 / 扩写 / 缩写
- 点击后调用 AI，弹出结果面板（带"插入编辑"和"复制"按钮）
- 不再打断写作流程
- 新增文件：`src/components/editor/SelectionToolbar.tsx`

### 8. 侧栏排序 (B2)

**问题：** 侧栏章节/卷只能按创建顺序排列，无法手动调整。

**修复：** 章节和卷各添加 ↑ ↓ 按钮（hover 可见），使用 `reorder_chapters` / `reorder_volumes` 命令实现排序。
- 修改文件：`WorkTree.tsx`
